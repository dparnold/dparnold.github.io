# Dominik Arnold Personal Website

This repository contains the static Jekyll site for `www.domarn.de`. It is built for GitHub Pages and deployed through GitHub Actions.

## Project Structure

- `index.md` contains the homepage profile data and hero text.
- `_sections/*.md` contains the content cards rendered below the hero.
- `_layouts/default.html` defines the page layout.
- `assets/css/site.css` contains the visual styling and responsive rules.
- `assets/img/` contains image assets used by the site.
- `_config.yml` contains the Jekyll configuration.
- `.github/workflows/deploy.yml` builds and deploys the site to GitHub Pages.

Do not edit files in `_site/` directly. That directory is generated by Jekyll.

## Requirements

Install Ruby and Jekyll before building locally.

```sh
gem install jekyll
```

On macOS, Ruby gems may install executables into a user gem directory that is not on `PATH`. If `jekyll` is installed but the command is not found, check:

```sh
gem env
```

Then add the reported executable directory to your shell `PATH`, or run Jekyll with the full executable path.

## Build Locally

From the repository root:

```sh
jekyll build
```

This generates the static site in `_site/`.

## Preview Locally

From the repository root:

```sh
jekyll serve
```

Jekyll will print a local preview URL, usually:

```text
http://127.0.0.1:4000/
```

Open that URL in a browser to preview the site.

If the `jekyll` command is not on `PATH`, use the installed executable directly. For example:

```sh
/Users/dominik/.gem/ruby/2.6.0/bin/jekyll build
/Users/dominik/.gem/ruby/2.6.0/bin/jekyll serve
```

## Editing Content

Update homepage profile fields in `index.md`.

Update the lower homepage sections in `_sections/`. Each section uses front matter like:

```yaml
---
title: Focus Areas
eyebrow: Work
anchor: work
order: 2
---
```

The `order` field controls the section order. The `anchor` field controls the HTML `id` used for navigation links.

## Deployment

Deployment is handled by `.github/workflows/deploy.yml`.

On every push to `main` or `master`, GitHub Actions:

1. Checks out the repository.
2. Builds the Jekyll site into `_site/`.
3. Uploads `_site/` as the GitHub Pages artifact.
4. Deploys it to GitHub Pages.

In the GitHub repository settings, Pages should use **GitHub Actions** as the source.

## Git Notes

The generated `_site/` directory is ignored and should not be committed. Commit source files such as Markdown, layout, CSS, config, and image assets instead.
